# 图论基础（最短路/MST/最大流/TSP）_图论网络类_MCM-B/ICM-D

## 1. 模型基础信息

- 【模型类别】图论网络类
- 【题型适配】MCM-B，ICM-D
- 【任务类型】路径规划，网络设计，流量分配，组合优化
- 【数据特征】网络数据（节点-边）；边权（距离/时间/成本/容量）
- 【触发关键词】最短路径、路径规划、最小生成树、最大流、容量、网络流、TSP
- 【创新方向】约束工程化，模型融合

## 2. 核心原理（AI 极简版）

把问题抽象为图：用最短路找成本最小路径，用 MST 以最小代价连通网络，用最大流刻画容量极限，用 TSP 处理“遍历所有点的最短回路”。

## 3. 美赛适用场景

- **最短路**：导航、应急路径、物流单车路线（Dijkstra/A*）。  
- **MST**：铺设管线/电缆/基站连接（总成本最小）。  
- **最大流**：运输/供给网络的最大吞吐、瓶颈识别。  
- **TSP**：巡检/配送/旅行路线（常配合 GA/SA）。  
- **Python 实现入口**：`networkx`（`shortest_path`/`minimum_spanning_tree`/`maximum_flow`/`approximation.traveling_salesman_problem` 等）。

## 4. 美赛创新适配方案（可直接用）

1) 【约束工程化】把时间窗/容量/优先级写入边权或约束（多段路径、分层网络）。  
2) 【融合】预测需求（Prophet/XGBoost）→ 动态调整边权/容量 → 再做最短路/流量优化。  
3) 【鲁棒性】对边权扰动/节点失效做敏感性，输出“关键边/关键节点”。

## 5. 优缺点与避坑指南

- **优点**：抽象清晰；可视化强；MCM-B 高频且拿分稳。
- **缺点**：真实约束多时，简单图论模型可能不够。
- **避坑**：不要把 TSP 当最短路；若是多车/时间窗问题，需升级到 VRP（可用启发式或 MILP 近似）。

## 6. 替代模型推荐

- MILP：网络设计/流量分配可线性化且规模可控时。
- GA/SA/ACO：大规模 TSP/VRP 的常用近似解法。
- 仿真（DES）：物流系统含排队/服务过程时。
