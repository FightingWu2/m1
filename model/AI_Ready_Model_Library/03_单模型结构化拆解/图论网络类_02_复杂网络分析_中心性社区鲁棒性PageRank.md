# 复杂网络分析（中心性/社区发现/鲁棒性/PageRank）_图论网络类_MCM-B/ICM-F

## 1. 模型基础信息

- 【模型类别】图论网络类
- 【题型适配】MCM-B，ICM-F
- 【任务类型】网络结构洞察，关键节点识别，团体/社区发现，网络韧性评估
- 【数据特征】网络数据；节点/边属性可用
- 【触发关键词】中心性、关键节点、社区发现、Louvain、鲁棒性、攻击、PageRank、影响力
- 【创新方向】验证与可解释，鲁棒/不确定性

## 2. 核心原理（AI 极简版）

通过中心性衡量节点重要性，通过社区发现识别群体结构，通过鲁棒性实验评估网络对故障/攻击的承受能力，PageRank 则用随机游走度量“权威性/影响力”。

## 3. 美赛适用场景

- **典型任务**：交通/社交/供应链网络的关键节点识别；传播干预策略；政策资源投放点选择。
- **数据条件**：需要构图规则（节点是什么、边是什么）；可以做静态网络或动态网络对比。
- **Python 实现入口**：`networkx`（中心性/`pagerank`）；社区发现可用 Louvain（环境允许时）或替代算法。

## 4. 美赛创新适配方案（可直接用）

1) 【鲁棒性分析】随机攻击 vs 蓄意攻击（按中心性排序移除）对比连通性变化，D 题常见高分点。  
2) 【干预策略】把“优先干预节点”定义为中心性前 k，模拟传播下降或网络效率提升。  
3) 【多指标综合】多种中心性做组合评分（或用熵权/组合赋权），避免单指标偏差。

## 5. 优缺点与避坑指南

- **优点**：图表表现力强；叙事贴合“关键节点/资源投放”；评审偏爱。
- **缺点**：构图主观性强；不同中心性结论可能不一致。
- **避坑**：必须解释构图依据；必须对比多种中心性并说明为何选最终指标；必须做鲁棒性实验支撑结论。

## 6. 替代模型推荐

- ABM：需要把网络结构放进传播仿真时。
- 最短路/最大流：更偏“优化”而非“结构洞察”时。
- 统计/ML：如果重点是节点属性预测而非结构分析。
