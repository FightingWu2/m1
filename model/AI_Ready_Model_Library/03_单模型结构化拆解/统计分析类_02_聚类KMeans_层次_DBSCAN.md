# 聚类（K-Means/层次聚类/DBSCAN）_统计分析类_MCM-C

## 1. 模型基础信息

- 【模型类别】统计分析类
- 【题型适配】MCM-C
- 【任务类型】分类分群（无监督），模式识别
- 【数据特征】无监督数据；距离度量可定义；可能存在噪声点（DBSCAN）
- 【触发关键词】分群、聚类、无监督、KMeans、层次聚类、DBSCAN、噪声点
- 【创新方向】验证与可解释，模型融合

## 2. 核心原理（AI 极简版）

K-Means 通过迭代最小化类内距离分群；层次聚类用聚合/分裂形成树状结构；DBSCAN 用密度把噪声点剔除并发现任意形状簇。

## 3. 美赛适用场景

- **典型任务**：城市/用户/区域分群；先分群再分别建模与评价（高频套路）。
- **数据条件**：特征需标准化；K 值要解释（肘部法/轮廓系数）；DBSCAN 需选 eps/min_samples。
- **Python 实现入口**：`sklearn.cluster.KMeans/AgglomerativeClustering/DBSCAN`；树状图用 `scipy.cluster.hierarchy`。

## 4. 美赛创新适配方案（可直接用）

1) 【聚类+评价】先聚类分群，再对每群做 TOPSIS/DEA/AHP，输出差异化策略。  
2) 【聚类有效性】轮廓系数/Calinski-Harabasz 指标对比，提升严谨性。  
3) 【可解释分群】用 PCA/t-SNE 可视化 + 各簇特征均值画像，形成“洞察叙事”。

## 5. 优缺点与避坑指南

- **优点**：无监督场景利器；与评价/预测融合很容易拿分。
- **缺点**：对尺度敏感；K-Means 假设球形簇。
- **避坑**：必须标准化；必须解释 K 或 DBSCAN 参数；不要把聚类结果当“真实标签”。

## 6. 替代模型推荐

- GMM：需要软聚类/概率解释时（更复杂）。
- 分类模型（SVM/RF/XGBoost）：有标签时不应使用无监督。
- 因子分析：更强调潜结构而非分群时。
