# 关联规则（Apriori）_统计分析类_MCM-C

## 1. 模型基础信息

- 【模型类别】统计分析类
- 【题型适配】MCM-C
- 【任务类型】模式挖掘（频繁项集/规则），行为洞察
- 【数据特征】事务型数据（集合/购买篮/事件集合）
- 【触发关键词】频繁项集、支持度、置信度、提升度、关联规则、推荐
- 【创新方向】验证与可解释

## 2. 核心原理（AI 极简版）

先挖掘频繁项集，再生成满足阈值的规则 \(A \rightarrow B\)，用支持度/置信度/提升度衡量规则强度。

## 3. 美赛适用场景

- **典型任务**：用户行为/事件共现模式、推荐规则、风险组合识别。
- **数据条件**：需要把数据整理成事务集合；阈值选择要解释。
- **Python 实现入口**：`mlxtend.frequent_patterns.apriori` + `association_rules`。

## 4. 美赛创新适配方案（可直接用）

1) 【分群后挖掘】先聚类分群，再在每群内部挖关联规则，得到“分群差异化洞察”。  
2) 【规则筛选】用提升度过滤伪相关，并做抽样验证规则稳定性。  
3) 【可视化】画规则网络图（节点=项，边=规则），提升展示效果。

## 5. 优缺点与避坑指南

- **优点**：结果可解释；适合做“洞察”而非预测。
- **缺点**：高维事务会爆炸；阈值选择影响巨大。
- **避坑**：别堆一堆规则不解释；必须给筛选标准与代表性规则。

## 6. 替代模型推荐

- XGBoost：如果目标是预测而非规则洞察。
- 聚类：先分群更能结构化洞察。
- 相关分析：变量是数值型而非事务集合时。
