# 熵权法（EWM）_评价类_MCM-C/ICM-F

## 1. 模型基础信息

- 【模型类别】评价类
- 【题型适配】MCM-C，ICM-F
- 【任务类型】赋权建模（客观权重）
- 【数据特征】有指标数据；指标波动差异显著
- 【触发关键词】客观赋权、信息熵、离散度、标准化、权重
- 【创新方向】模型融合，验证与可解释

## 2. 核心原理（AI 极简版）

指标越“有区分度”（离散度越大），信息熵越小，权重越大；用数据本身决定权重而非主观打分。

## 3. 美赛适用场景

- **典型任务**：多指标评价前的客观赋权；与 TOPSIS/灰色关联配套使用。
- **数据条件**：需要标准化（正向/逆向指标处理清楚）；避免零值导致对数问题（加小常数）。
- **Python 实现入口**：`numpy/pandas` 标准化 + 熵值计算 + 权重输出。

## 4. 美赛创新适配方案（可直接用）

1) 【组合赋权】与 AHP 做博弈论组合赋权（主客观融合），显著提升说服力。  
2) 【稳健性】用不同标准化方法（min-max / z-score）对比权重稳定性。  
3) 【解释性】报告“权重贡献”与“指标区分度”，避免只给一串数字。

## 5. 优缺点与避坑指南

- **优点**：客观、可复现；适合数据驱动的 MCM-C。
- **缺点**：若指标波动小或噪声大，权重可能不合理。
- **避坑**：必须先定义正/逆向指标；缺失值要处理；否则权重会失真。

## 6. 替代模型推荐

- AHP：无数据或需专家权重时。
- PCA：指标高度相关且需降维时（先 PCA 再评价）。
- GRA：小样本且希望做关联度排序时。
