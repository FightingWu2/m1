# 组合评价（熵权-TOPSIS / 灰色-TOPSIS）_评价类_MCM-C/ICM-F

## 1. 模型基础信息

- 【模型类别】评价类
- 【题型适配】MCM-C，ICM-F
- 【任务类型】排序评估，综合评分，方案优选
- 【数据特征】多指标数据；小样本/不确定性时更偏向灰色-TOPSIS
- 【触发关键词】组合评价、客观赋权、灰色关联、鲁棒排序、小样本
- 【创新方向】模型融合，鲁棒/不确定性

## 2. 核心原理（AI 极简版）

先解决“权重怎么来”（熵权），再解决“怎么排”（TOPSIS）；当数据不完备/样本少时，用灰色关联度增强排序鲁棒性。

## 3. 美赛适用场景

- **典型任务**：多方案优选（政策/工程/管理）；数据指标较多但样本不大。
- **数据条件**：标准化与正/逆向指标处理明确；灰色部分要说明参考序列与分辨系数。
- **Python 实现入口**：`numpy/pandas`（熵权 + TOPSIS 计算）；灰色关联系数可自写。

## 4. 美赛创新适配方案（可直接用）

1) 【熵权-TOPSIS 作为主线】给出权重与贴近度，并解释关键指标贡献。  
2) 【灰色修正】对小样本/噪声场景，用 GRA 修正距离或并行输出“灰色贴近度”。  
3) 【一致性验证】用不同标准化方法/分辨系数做对比，展示排序稳定性。

## 5. 优缺点与避坑指南

- **优点**：比单一 TOPSIS 更稳健；“有创新但不复杂”，符合美赛偏好。
- **缺点**：参数（分辨系数等）需要说明；组合过多可能显得堆叠。
- **避坑**：别把组合写成“拼盘”；要说明每一步解决了哪个痛点（权重偏差/小样本不确定）。

## 6. 替代模型推荐

- DEA：投入-产出效率场景更合适。
- Fuzzy：评价语言模糊且需隶属度时。
- PCA：指标高度相关，需先降维再评价时。
