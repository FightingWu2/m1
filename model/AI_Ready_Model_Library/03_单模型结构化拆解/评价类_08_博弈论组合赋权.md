# 博弈论组合赋权（AHP+熵权等）_评价类_MCM-C/ICM-F

## 1. 模型基础信息

- 【模型类别】评价类
- 【题型适配】MCM-C，ICM-F
- 【任务类型】赋权建模（主客观融合），排序评估
- 【数据特征】既有数据也有专家判断；希望避免单一权重偏差
- 【触发关键词】组合赋权、博弈论、主客观融合、权重一致性、鲁棒
- 【创新方向】模型融合，验证与可解释

## 2. 核心原理（AI 极简版）

把不同赋权方法（AHP、熵权等）得到的权重向量视为“参与者”，通过博弈论思想求一个折中组合权重，减少偏差。

## 3. 美赛适用场景

- **典型任务**：政策/管理评价中既要“专家合理性”又要“数据客观性”的权重确定。
- **数据条件**：至少两套权重来源（如 AHP + EWM）；指标体系需一致。
- **Python 实现入口**：`numpy` 线性代数求组合系数（按选定的组合规则实现）。

## 4. 美赛创新适配方案（可直接用）

1) 【主客观闭环】AHP 确立逻辑层次，EWM 把数据差异写进权重，再组合输出最终权重。  
2) 【稳健性】对每套权重做扰动或换标准化方式，观察组合权重排序稳定。  
3) 【解释性】说明“为什么单一方法会偏”，以及组合后如何改善（给对比表/图）。

## 5. 优缺点与避坑指南

- **优点**：评审常认为这是“高阶但不复杂”的加分项；提升可信度。
- **缺点**：若组合规则不说明，会被质疑“黑箱拼权重”。
- **避坑**：必须写出组合规则与约束（系数非负、和为1等）；必须给对比（单一 vs 组合）。

## 6. 替代模型推荐

- 单一 AHP：纯主观、无数据时。
- 单一 EWM：纯数据驱动、无需专家时。
- PCA：指标共线严重，先降维再评价时。
