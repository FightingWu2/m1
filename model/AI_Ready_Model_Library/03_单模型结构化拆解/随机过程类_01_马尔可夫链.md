# 马尔可夫链（Markov Chain）_随机过程类_MCM-C/ICM-F

## 1. 模型基础信息

- 【模型类别】随机过程类
- 【题型适配】MCM-C，ICM-F
- 【任务类型】状态转移预测，长期稳态分析
- 【数据特征】离散状态；可估计转移概率矩阵
- 【触发关键词】状态转移、转移矩阵、稳态分布、长期比例、无记忆性
- 【创新方向】验证与可解释，鲁棒/不确定性

## 2. 核心原理（AI 极简版）

下一时刻状态只依赖当前状态，通过转移矩阵的幂得到长期分布与多步预测。

## 3. 美赛适用场景

- **典型任务**：市场份额转移、用户状态演化、系统状态切换（政策/管理题常见）。
- **数据条件**：状态定义清晰；有历史转移数据可估计转移概率（或用专家假设并解释）。
- **Python 实现入口**：`numpy` 矩阵归一化 + `matrix_power`/特征向量求稳态。

## 4. 美赛创新适配方案（可直接用）

1) 【吸收马尔可夫链】定义吸收态（“成功/失败/退出”）并计算吸收概率与期望时间。  
2) 【不确定性】对转移概率做区间扰动或贝叶斯估计，输出稳态区间。  
3) 【与蒙特卡罗融合】用蒙特卡罗模拟多条轨迹验证解析结果与置信区间。

## 5. 优缺点与避坑指南

- **优点**：结构清晰、可解释；适合“状态型叙事”的政策/管理题。
- **缺点**：状态划分主观；“无记忆性”假设可能不成立。
- **避坑**：状态不能太多（稀疏转移会不稳定）；必须解释状态划分与转移来源。

## 6. 替代模型推荐

- ARIMA/Prophet：更偏连续数值时间序列而非离散状态。
- HMM：如果需要“隐状态”（更复杂，可作为创新但实现成本高）。
- 系统动力学/ABM：如果过程依赖历史或个体交互更强。
