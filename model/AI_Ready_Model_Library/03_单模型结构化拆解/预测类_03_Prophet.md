# Prophet_预测类_MCM-C/ICM-E/ICM-F

## 1. 模型基础信息

- 【模型类别】预测类
- 【题型适配】MCM-C，ICM-E，ICM-F
- 【任务类型】预测回归（时间序列），趋势分解
- 【数据特征】时序数据；缺失值/异常点较多；节假日/事件效应明显
- 【触发关键词】节假日、事件冲击、缺失、趋势分解、季节性、分段趋势
- 【创新方向】验证与可解释，鲁棒/不确定性

## 2. 核心原理（AI 极简版）

把序列分解为“趋势 + 季节性 + 节假日/事件项”，用可解释的加法模型拟合，并可输出预测区间。

## 3. 美赛适用场景

- **典型任务**：政策/环境/社会系统的时间序列预测（ICM-E/F 常见），且数据不干净。
- **数据条件**：需要日期列；节假日/事件可构造特征（没有也可用默认）。
- **Python 实现入口**：`prophet`（原 `fbprophet`）。

## 4. 美赛创新适配方案（可直接用）

1) 【事件建模】把政策发布/灾害/管控等写成假日或回归项，显著提升解释力。  
2) 【不确定性】输出预测区间并解释“风险范围”，用于决策建议。  
3) 【对比基线】与 ARIMA/指数平滑对比，证明选择 Prophet 的理由（缺失/异常鲁棒）。

## 5. 优缺点与避坑指南

- **优点**：对缺失/异常鲁棒；趋势与季节性解释性强；实现快。
- **缺点**：对强非线性复杂模式不如深度学习；参数理解需要一点经验。
- **避坑**：日期频率要一致；不要用 Prophet 去预测无时间结构的数据。

## 6. 替代模型推荐

- ARIMA/SARIMA：数据较干净且希望统计检验时。
- LSTM/GRU：数据量大且非线性强时。
- XGBoost：把时间特征工程化后做监督学习时。
