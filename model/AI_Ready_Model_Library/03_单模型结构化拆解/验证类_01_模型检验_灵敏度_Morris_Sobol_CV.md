# 模型检验与灵敏度分析（含 Morris/Sobol、交叉验证）_统计分析类_全题型通用

## 1. 模型基础信息

- 【模型类别】统计分析类
- 【题型适配】全题型通用
- 【任务类型】验证与可解释，风险不确定
- 【数据特征】任何模型结果都需要可靠性证明
- 【触发关键词】灵敏度分析、稳健性、参数扰动、Sobol、Morris、交叉验证、泛化
- 【创新方向】验证与可解释，鲁棒/不确定性

## 2. 核心原理（AI 极简版）

通过改变关键参数/数据划分方式，观察结论是否稳定；机器学习用交叉验证评估泛化，机理/优化模型用灵敏度分析评估稳健。

## 3. 美赛适用场景

- **任何题**：没有敏感性/验证的论文很难拿高奖（评审硬性期待）。
- **Python 实现入口**：  
  - CV：`sklearn.model_selection.KFold/cross_val_score`  
  - Sobol/Morris：若环境允许可用 `SALib`；否则可用“参数扫面 + 统计”实现简化版。

## 4. 美赛创新适配方案（可直接用）

1) 【局部敏感性】关键参数 ±5%/±10% 扰动，画结果变化率条形图（必做）。  
2) 【全局敏感性】Morris/Sobol 输出全局重要性排序（高级加分）。  
3) 【验证闭环】用留出集/滚动验证/仿真对照验证，把“模型可信”写成证据链。

## 5. 优缺点与避坑指南

- **优点**：直接提升可信度与获奖概率；能解释“结论稳不稳”。
- **缺点**：需要额外计算与组织实验。
- **避坑**：不要只做一句“我们做了敏感性分析”；必须给图表与结论；时间序列不要随机打乱做 CV。

## 6. 替代模型推荐

- 只做情景分析：时间极紧时的最低保底（但仍建议给 ±10% 扰动）。
- 蒙特卡罗：对不确定参数做分布抽样更全面。
