# AI 调用测试报告（Prompt → 期望输出 → 校验点）

> 目的：验证 AI 能否在“题型 + 任务关键词 + 数据特征”输入下，稳定输出：  
> **1-2 个 Baseline + 1 个 Innovation/融合**，并能指向对应模型卡片与实现入口。

## 测试用例 1：MCM-B 城市物流路径优化（框架示例）

- **输入指令**：请根据 MCM-B 题“城市物流路径优化”任务，匹配最优模型及创新方案。  
- **期望匹配**：  
  - Baseline：[`图论基础（最短路/MST/最大流/TSP）`](03_单模型结构化拆解/图论网络类_01_图论基础_最短路MST最大流TSP.md)（★★★）  
  - Baseline：[`MILP/0-1`](03_单模型结构化拆解/优化类_02_MILP与0-1规划.md)（★★★，若规模可控）  
  - Innovation：[`GA`](03_单模型结构化拆解/优化类_07_遗传算法GA.md) + [`SA`](03_单模型结构化拆解/优化类_10_模拟退火SA.md)（★★★，混合精修）  
- **校验点**：  
  - 是否明确触发词：TSP/路径/容量/时间窗/车辆数  
  - 是否给出实现入口：`networkx` + `deap/pygad`（或自写 GA/SA）  
  - 是否包含验证：多次运行统计 + 收敛曲线 + 参数敏感性

## 总结：当前库的通过标准

- AI 能稳定给出：题型判定 → 标签匹配 → Baseline+Innovation → 实现入口 → 验证/敏感性。  
- 若出现误匹配：优先回到 `02_AI适配标签规范表.md`，补充触发关键词或收敛“任务类型”枚举，再更新对应模型卡片关键词块。
