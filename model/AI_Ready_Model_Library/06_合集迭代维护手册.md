# 合集迭代维护手册（年度更新规则 + 维护清单）

> 目标：每年美赛结束后，快速把“高频模型 + 获奖创新点”沉淀进库，保持 AI 匹配的命中率与可落地性。

## 1. 版本规则

- **版本号**：`YYYY.MCM.v{n}`（例：`2026.MCM.v1`）
- **变更记录**：每次发布在本文件末尾追加 Changelog（新增/修改/弃用）。
- **兼容性原则**：受控标签（见 `02_AI适配标签规范表.md`）只增不改；如需改名，保留旧标签别名。

## 2. 年度迭代流程（建议每年赛后 2 天内完成）

1) **收集材料**：挑选获奖论文（尤其 O/Finalist）中高频方法与创新段落。  
2) **更新模型清单**：补充到 `01_模型原始清单_去重分类.md`，并标注“新增/复用/弃用”。  
3) **补齐模型卡片**：在 `03_单模型结构化拆解/` 新增对应卡片，必须满足 6 模块模板与受控标签。  
4) **更新匹配矩阵**：在 `04_美赛题型-模型智能匹配矩阵.md` 更新 ★优先级（依据“高频 + 可复现 + 写作友好”）。  
5) **回归测试**：用 `05_AI调用测试报告.md` 的用例跑一遍（人工检查 AI 输出是否命中）。  
6) **发布最终版入口**：更新 `MCM_模型算法合集_AI适配版_最终版.md` 的链接与说明。

## 3. 维护清单（必须检查）

- **关键词扩充**：加入当年新热点词（例：碳中和、数字孪生、韧性、极端天气、供应链冲击等）。  
- **模型弃用**：仅当模型“难复现/不落地/几乎不再出现”才弃用，并保留替代建议。  
- **库函数更新**：确认关键库名称/入口函数是否变更（如 `fbprophet → prophet`）。  
- **叙事模板**：创新优先写“融合 + 验证”，避免堆砌复杂名词。

## 4. 质量门槛（评审视角）

新增/修改任何模型卡片必须满足：

- **可匹配**：触发关键词 ≥ 5，且覆盖任务词 + 数据词（必要时加约束词）。
- **可落地**：给出明确 Python 入口（库/函数或“自写实现点”）。
- **可验证**：明确至少 1 个验证手段（CV/残差检验/仿真对照/灵敏度）。
- **可替代**：给出至少 2 个替代模型（当假设不满足时能快速切换）。
